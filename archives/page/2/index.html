豆瓣电台API分析

前几天看到命令行版的电台cmd.fm，表示键盘党最爱。 可惜音乐都是来自sound cloud，大天朝的重口味和网速各种水土不服，于是首先想到的就是要山寨个用豆瓣电台音乐的版本。 豆瓣毫无悬念的没有公开API，不过这个完全不影响抓包分析，完整的文档放在GitHub Douban FM API Documentation。

现在已经用这个API做了一个命令行风格豆瓣电台的Chrome插件，FM Terminal。

所有的音乐都是防盗链的，所以Web应用的可能性不大。略诡异的是放出红心的概率比较低，不像官方版的那么容易陷入红心无法自拔。

其中比较有意思的是两点，一是广告很容易和其它普通、二逼或者文艺的音乐区分开来。 另一个是豆瓣追踪用户播放行为的方式，正常播放完成的音乐豆瓣不关心，只对加红心、丢垃圾桶和跳过这样的“异常”行为感兴趣。传回去的记录是时序的，说明后台的算法应该是和时序有关的。

不过我只想说，敢不敢让广告也能加心拖垃圾桶，挖掘下用户爱看神马广告。

GPU并行编程CUDA5.0+Visual Studio 2012环境配置

问题

CUDA5.0并不支持Visual Studio 2012的v110 toolchain，所以在安装后尝试编译，会先后遇到如下的错误：

nvcc : fatal error : '-ccbin': expected a number
**nvcc fatal : nvcc cannot find a supported cl version. Only MSVC 8.0 and MSVC 9.0 are supported**

解决方案：

1.部署/修改配置文件

在C:\Program Files (x86)\MSBuild\Microsoft.Cpp\v4.0\V110\BuildCustomizations文件夹中是MSBuild要完成编译的自定义配置文件，需要手动部署（拷贝）到C:\Program Files (x86)\MSBuild\Microsoft.Cpp\v4.0\V110\BuildCustomizations文件夹下。本文中路径为Win7 64bit默认安装后的路径，根据自己的安装情况调整。

在这之前需要修改如下文件：

CUDA5.0.props中

找到如下行：

<CudaClVersion Condition="'$(PlatformToolset)' == 'v90'">2008</CudaClVersion>
<CudaClVersion Condition="'$(PlatformToolset)' == 'v100'">2010</CudaClVersion>
添加： 
<CudaClVersion Condition="'$(PlatformToolset)' == 'v110'">2010</CudaClVersion>
**CUDA 5.0.targets中** 找到CudaCleanDependsOn标签，改为： 
<CudaCleanDependsOn>
  $(CudaCompileDependsOn);
  _SelectedFiles;
  CudaFilterSelectedFiles;
  AddCudaCompileMetadata;
  AddCudaLinkMetadata;
  AddCudaCompileDeps;
  AddCudaCompilePropsDeps;
  ValidateCudaBuild;
  ValidateCudaCodeGeneration;
  ComputeCudaCompileOutput;
  PrepareForCudaBuild
</CudaCleanDependsOn>
找到GenerateRelocatableDeviceCode，改为 
GenerateRelocatableDeviceCode="%(CudaCompile.GenerateRelocatableDeviceCode)"
找到CodeGeneration，改为 
CodeGeneration="%(CudaCompile.CodeGenerationValues)"
找到CommandLineTemplate，改为 
CommandLineTemplate=""$(CudaToolkitNvccPath)" %(CudaCompile.BuildCommandLineTemplate) %(CudaCompile.ApiCommandLineTemplate) %(CudaCompile.CleanCommandLineTemplate)" />

2. 修改头文件

C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v5.0\include文件夹中，host_config.h文件。

找到

#if _MSC_VER < 1400 || _MSC_VER > 1600

#error -- unsupported MSVC version! Only MSVC 8.0, 9.0, and 10.0 are supported!
修改为： 
#if _MSC_VER < 1400 || _MSC_VER > 1700

#error -- unsupported MSVC version! Only MSVC 8.0, 9.0, and 10.0 are supported!

3. 语法高亮支持

  1. 菜单“Tools->Options…”，展开左侧“Text Editor”，点击“File Extension”.
  2. 在“Extension”框中输入“cu”, 选择“Microsoft Visual C++”，点击“Add”，捅“OK”。
  3. 重新打开文件

用批处理在Windows下实现sudo命令

在Windows下运行某些命令的时候，必须要管理员权限。每次要重新选择“以管理员权限运行”很不方便而且没有效率，于是写了一个批处理文件，实现类似Linux下的sudo命令。

保存为sudo.bat，放在环境变量PATH中指定的任意一个文件夹下，在命令行窗口中输入：

sudo [command line]

就会出现UAC对话框，确认后就会以管理员权限执行指定的命令。

在Hec的实现中，有一个基本的功能，就是实现虚拟机字节码的endian与native code的endian之间的转换（如果相反），只是简单的颠倒字节顺序。主要在虚拟机加载字节码，开始执行之前调用，直接影响字节码加载到开始执行期间用户的等待时间。
